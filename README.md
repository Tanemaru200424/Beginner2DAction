# けーびろぼ！
## 概要
2Dジャンプアクション。主人公「けーびろぼ」を操作して兵器工場を乗っ取った黒幕を撃破しよう。  
アクションは歩き、ジャンプ、攻撃の3種類。攻撃は溜めることができ、最大チャージすれば倒した敵や飛び道具を反射したり、特定行動中のボスをダウンさせたりできるぞ。  
現在はチュートリアルのアクションシーンしかできておらず、タイトルやエンディング、ステージの続きは未実装です。

## 工夫した点
* ダメージ処理を担うインターフェースIDamageableと、最大溜め攻撃を受けた時の処理IHittableを実装し、ダメージと吹き飛ばしの処理を分離しました。
* 独自のY軸速度計算により自然な加速を表現しました。はじめは上昇、落下を等速にしていましたが、急な速度変更はプレイヤーにとってストレスになると考えこのようにしました。
* 制御が複雑なオブジェクト（自機とボス）は状態管理、物理挙動、攻撃、ダメージ等責任を細かく分けたスクリプトにより実装しました。
* 複数の状態を持つオブジェクトはEnum型の変数を切り替えることで制御しました。
* 攻撃モーションや登場退場演出といったアニメーションが終わってから状態を変更する必要がある場合、SubStateMachineを用いてアニメーション終了を検知するようにしました。
* フェードインフェードアウトを利用して、ステージの地形とエリア切り替えやプレイヤーの位置変更をプレイヤーに見せないようにしました。一方、カメラスクロールの際は敵や攻撃オブジェクトが消える瞬間をプレイヤーに見せ、エリアが切り替わったことを伝えるようにしました。
* アクション中動的に生成されるオブジェクトを1つのコンテナに登録することで、一時停止やオブジェクトの破壊等を管理しやすくしました。

## プロジェクト構成
以下は一部省略したものです。
```
Assets/
└ Scripts/                # スクリプトは共通の置き場にして場所を分かり易く。
  ├ ActionObjects/        # プレイヤー、敵、エフェクトなどアクションシーンに登場するオブジェクトの実装。
  | ├ Effect/             # エフェクトの制御。
  | ├ Enemy/              # 敵と攻撃オブジェクトの制御。
  | ├ Gimmick/            # ギミックの制御。
  | ├ GroundChecker/      # ギミックやキャラクター等多くのオブジェクトで使いそうな地形検知系の実装
  | ├ ObjectAttribute/    # オブジェクト識別ラベルやプレイヤー追尾等の実装。GimmickやEnemyとの境界が曖昧。
  | └ Player/             # プレイヤーと攻撃オブジェクトの制御。
  ├ ActionScenesSystems/  # アクションシーンのシステム周りの実装。
  | ├ GameFlow/           # ゲーム開始やリスタート、ボス演出等の実装。
  | ├ ObjectManagers/     # 一時停止、オブジェクト破棄、プレイヤー追尾へのターゲット登録等の実装。
  | └ StagePartsManagers/ # ステージの部品制御。ギミックの一斉起動、停止等。
  └ UI/                   # フェード、一時停止で使う一枚絵の切り替えやプレイヤー、ボスの情報表示。
```

## 操作方法（PC / ゲームパッド）
### キャラクター操作
まだテストプレイが不十分なため操作しにくい配置になってしまっています。
* 移動：AキーまたはDキー / Dパッド左右
* ジャンプ：Wキー / Southボタン
* 攻撃：Fキー / Westボタン
* 吹き飛ばし方向制御：↑キーまたは↓キー / R1ボタンまたはR2ボタン
* 一時停止：Spaceキー / Startボタン
### デバッグ用機能
クリアとゲームオーバーが重なった際のイベント進行制御の実装のためにある機能です。
強制ダメージ：Zキー
